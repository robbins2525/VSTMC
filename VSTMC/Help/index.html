Visual Studio Tools for MicroStation CONNECT Edition

Visual Studio Tools for MicroStation CONNECT Edition integrates project templates, item templates, snippets, schema support, and productivity commands into Visual Studio to streamline Bentley add-in development across .NET and native workflows.

Introduction

VSTMC is an extension for Bentley-focused developers who build MicroStation CONNECT Edition add-ins and related tools in C#, Visual Basic, and native C++. It packages common project scaffolding, command table support, key-in helpers, schemas, snippets, and convenience tools into one development companion.

The original CHM help system organized topics around templates, tools, and setup tasks. This HTML edition keeps the same spirit, but makes the content easier to maintain, update, and browse in a modern browser.

What’s New in This Edition

  • Documentation is transitioning from legacy CHM help to HTML help.
  • Minimum supported Visual Studio is now Visual Studio 2022.
  • Bentley Help topics can be surfaced from the Help menu as a proper submenu of clickable items.
  • Templates and generated code are being modernized for cleaner C# and VB patterns.
  • Nullable-aware C# patterns may be used where project configuration supports them.

Supported Bentley Products

The legacy help included a dedicated page for supported Bentley products. In practice, support depends on the project template selected, installed SDKs, and available references for the target Bentley application.

Typical supported product families include MicroStation CONNECT Edition and related Bentley platform products such as OpenRoads, OpenRail, OpenBridge, OpenPlant, and others when the proper SDK components are installed.

Project Templates

Project templates provide the starting structure for Bentley add-in development. The original help included separate descriptions for C# and VB.NET project templates and how they mapped to Bentley applications.

Item Templates

Item templates are one of the biggest time-savers in VSTMC. They generate common Bentley development building blocks such as selection tools, placement tools, forms, WPF controls, primitive tools, command helpers, and scan criteria extensions.

Code Snippets

The legacy help documented snippet support for quickly inserting common code structures into Bentley add-in projects. Snippets can speed up repetitive tasks such as command wiring, tool setup, and helper method generation.

Commands.xml and Key-ins

Commands.xml remains a central piece of Bentley add-in development. VSTMC helps wire command handlers, key-ins, keywords, and other command table elements into the project structure.

  • Generate key-in handlers
  • Insert new command entries when templates are added
  • Keep command table structure aligned with generated code
  • Support IntelliSense and schema-driven editing

Key-in Tree

The key-in tree organizes command definitions and helps structure command hierarchies in a more readable way. This area of the original help covered the tree format and related schema support.

Schemas and IntelliSense

Extension Tools

The extension adds several productivity commands to Visual Studio. These tools reduce friction when working with Bentley SDKs, folders, forums, and help content.

Bentley Import Tool

Bentley Import Tool helps bring Bentley application content into the project workflow. The older help documented this as one of the main productivity tools in the extension.

This feature is especially useful when bridging managed and native Bentley development or when integrating existing native assets into a Visual Studio-based workflow.

Open Folder Commands

Folder-opening commands are convenience shortcuts that launch common Bentley-related folders in File Explorer. These are typically driven by environment variables and project context.

  • Open Bentley SDK Folder
  • Open MDLAPPS Folder
  • Open MSCE Folder

If these commands appear to do nothing, verify the environment variables and target folder paths used by the command logic.

Search Bentley Forums

This tool uses the current text selection in the editor to search Bentley Communities more quickly. It is helpful when error messages, class names, or API terms need a quick lookup in community discussions.

For better results, this workflow can also be paired with broader web search strategies that target Bentley Communities specifically, rather than relying on deep documentation URLs that may shift over time.

Bentley References

One of the quiet but important jobs performed by the extension is adding or shaping Bentley assembly references based on the selected project and Bentley product context.

Legacy help pages showed product-specific reference injection and project setup. This HTML edition should continue documenting those patterns, especially where product-specific references differ.

Help System

The VSTMC help system is now transitioning from CHM-based help to HTML help. HTML help is easier to maintain, safer to open, easier to style, and simpler to expand into multi-page documentation later.

Bentley Help items can also be surfaced in Visual Studio as a submenu of clickable CHM topics while the broader documentation system transitions to HTML.
